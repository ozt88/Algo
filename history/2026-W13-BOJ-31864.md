\uFEFF# 2026-W13-BOJ-31864

## 문제
- 출처: BOJ
- 번호: 31864
- 이름: 눈송이 탕후루 만들기
- 링크: https://www.acmicpc.net/problem/31864

## 요약
- 목표: (0,0)에서 끝점 (ex, ey)까지의 선분 위에 있는 과일을 최대한 많이 꽂기
- 제약: N, M ≤ 300,000 / 좌표 절댓값 ≤ 10^9
- 출력: 최대로 꽂히는 과일 수 (정수 1개)

## 핵심 알고리즘
- 방향 정규화: `gcd(|fx|, |fy|)`로 나눈 `(fx/gcd, fy/gcd)` pair를 map key로 사용
- 반대 방향 분리: `(-1,-2)`와 `(1,2)`는 다른 key — 꼬치가 (0,0)에서 시작하므로 의도된 동작
- 거리 비교: scale = gcd 값. `과일 scale ≤ 끝점 scale`이면 해당 꼬치에 포함
- 그룹화 및 카운트: `map[dir][]int` (정렬된 scale 배열) + `sort.SearchInts`로 O(log N) 카운트
- 전체 복잡도: 전처리 O(N log N), 쿼리 O(M log N)

## 실패/버린 아이디어
- 없음 (discuss-phase에서 사용자가 직접 올바른 방향으로 설계)

## 최종 선택 이유
- 사용자가 "pair를 key로 하는 map, 정렬된 배열로 scale 값 저장" 방식을 직접 설계
- "x,y의 GCD로 나눠서 unit vector를 key로" — 방향 정규화 핵심 아이디어
- 반대 방향 key 분리: "꼬치는 0,0에서 시작하니깐" 으로 직접 확인

## 구현 메모
- (0,0) 과일 처리: gcd(0,0)=0이 되어 division by zero 발생 가능 — 현재 구현은 별도 처리 없음 (문제 제약상 (0,0) 과일은 없는 것으로 판단)
- `sort.SearchInts(scales, g+1)`: scale ≤ g 인 과일 수를 O(log N)에 계산
- golangci-lint errcheck/gosec: `fmt.Fscan` 반환값에 `//nolint:errcheck,gosec` 주석으로 대응

## 테스트
- 예제 1: 9개 과일, 5개 끝점 → 3 (끝점 (4,4) 방향, dir(1,1) scale 1·2·3 포함)
- 단일과일단일끝점_일치: solve([[1,2]], [[1,2]]) == 1
- 방향불일치: solve([[1,2]], [[2,1]]) == 0
