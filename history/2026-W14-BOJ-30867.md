# BOJ 30867 과제가 너무 많아

## 문제 요약
- 길이 L의 문자열 S에 N번 두통 연산(wh → hw) 적용 결과를 출력하는 문제
- 두통 연산: 문자열에서 'w' 바로 다음에 'h'가 오는 첫 쌍을 swap (wh→hw)
- w와 h가 아닌 다른 문자는 장벽 역할 — 장벽을 넘어 swap 불가
- L, N ≤ 200,000 (int 범위 충분)
- 출력: N번 연산 후 최종 문자열

## 핵심 알고리즘
- **D-01 세그먼트 분리**: w/h가 아닌 문자는 장벽 → 문자열을 w/h로만 이루어진 세그먼트로 분리, 각 세그먼트 독립 처리
- **D-02 h의 이동 방향**: 각 세그먼트 내에서 h는 왼쪽으로 버블링. 한 번의 연산에 최대 1칸 이동
- **D-03 최종 위치 공식**: 각 h의 최종 위치 = 원래위치 - min(왼쪽_w개수, N)
  - 왼쪽_w개수 = 원래위치 - (이 h 앞의 h개수)
  - h끼리 충돌 없음: h_k의 새 위치 = pos_k - min(wLeft_k, N). wLeft_k ≥ wLeft_{k-1}이고 pos_k > pos_{k-1}이므로 순서 보존 수학적 증명
- **D-04 시간복잡도**: O(L) — N번 시뮬레이션 불필요
- **D-05 구현**: 세그먼트별 h 위치 수집 → 새 위치 계산 → 빈 자리 'w' 채움 → 원래 슬라이스에 복사

## 실패/버린 아이디어
- **N번 직접 시뮬레이션 O(L×N)**: 매 연산마다 wh 쌍을 찾아 swap하는 방식
  → N=200,000, L=200,000이면 최대 4×10^10 연산 → TLE 확실, 폐기
- **각 h를 개별 BFS/추적**: h마다 경로 추적하는 방식
  → 세그먼트 내 순서 보존 증명이 성립하므로 불필요, 단순 공식으로 대체

## 최종 선택 이유
- h끼리 순서가 보존된다는 수학적 증명으로 충돌 걱정 없이 각 h를 독립적으로 이동 가능
- 세그먼트 분리 후 단일 패스로 O(L) 달성 — 시간복잡도 최적
- 구현이 단순하고 엣지 케이스(h 없는 세그먼트, w 없는 세그먼트, 장벽만 있는 경우) 자연스럽게 처리됨
