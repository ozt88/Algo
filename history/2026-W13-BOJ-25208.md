\ufeff# BOJ 25208 새벽의 탐정 게임

## 문제 요약
- N×M 격자에서 탐정(D)이 한 면이 뚫린 정육면체 감옥을 굴려 도둑(R) 위치로 이동
- 뚫린 면이 바닥일 때 도둑 칸 도달 시 승리 / 막힌 면이 바닥이면 해당 이동 스킵
- 4 ≤ N, M ≤ 500, 테두리 전체 벽, 시간 1초, 메모리 1024 MB
- 출력: 최소 굴리기 횟수 (불가능 시 -1)

## 핵심 알고리즘
- **상태**: `(r, c, s)` — 격자 좌표 + 큐브 orientation
- **s 인코딩**: `s = north_face × 6 + bottom_face` (범위 0~35, 유효 상태 24개)
- **초기 큐브**: `{바닥:0, 윗:1, 북:2, 남:3, 동:4, 서}`  →  초기 s = 2×6+0 = **12**
- **전환 테이블**: 초기 배열에서 BFS로 24개 상태 × 4방향 96개 엔트리를 런타임에 자동 생성
- **BFS 탐색**: dp[r][c][s] = 최소 이동 횟수, 이미 저장값 이하면 스킵
- **R 칸 진입 조건**: bottom_face == 0(뚫린 면)이면 승리 반환, 아니면 해당 이동 스킵

## 실패/버린 아이디어
- **거리 parity 접근**: 최단 거리 + parity로 orientation을 추론하려 했지만 경로 의존성이 있어 불가 (기각)

## 최종 선택 이유
- 상태 공간 500×500×36 ≈ 9M (유효 6M)으로 BFS가 제한 시간 내에 충분히 동작
- 전환 테이블을 하드코딩하지 않고 BFS 자동 생성 → 실수 없이 24개 상태 전이 보장
- R 칸을 일반 이동과 다르게 처리(승리 조건 체크 후 스킵)하는 방식이 코드 흐름을 단순하게 유지

## 테스트 결과 및 AC 여부
- 예제 1 (5×5, D=(1,1) R=(3,3)) → 4 ✓
- 예제 2 (6×5, D=(4,2) R=(2,3)) → 7 ✓
- 예제 3 (4×6, D=(1,4) R=(1,1)) → -1 ✓
- 제출 결과: **AC**
