# BOJ 28280 귀납법

## 문제 요약
- 1에서 시작해 `×2` 또는 `-1` 연산으로 k를 만드는 최소 횟수
- 도중 값은 항상 양의 정수여야 함
- T ≤ 20, k의 총합 ≤ 4×10⁶, 시간 1초

## 실패/폐기한 아이디어
- **Top-down DP (K에서 역방향):** `DP(K) = min(DP(K/2), DP(K-1)) + 1` 로 시작
  - `-1` 연산의 역방향이 `+1`임을 간과 → 올바른 점화식은 `min(DP(K/2), DP(K+1)) + 1`
  - 그런데 `DP(K+1)`이 필요하면 `DP(K+2)` ... 무한히 위로 올라가는 문제 발생
  - Top-down DP 포기

## 선택한 방향과 이유
- **1에서 앞으로 BFS 전파:** dist 배열을 -1로 초기화 후, 1에서 출발해 `×2`와 `-1` 전이를 BFS로 전파
- k 총합이 4×10⁶이므로 배열 크기 4_000_001로 한 번에 전처리 후 쿼리마다 O(1) 조회
- `Wrong proof!`는 어떤 k든 2의 거듭제곱까지 올라간 뒤 `-1`로 내려오면 항상 도달 가능하므로 실제로 출력되지 않음

## 구현 메모
- 큐를 슬라이스 + head 인덱스로 구현 (재할당 방지)
- `×2` 전이는 `next < maxK` 조건으로 범위 초과 차단

## 테스트 및 결과
- k=1 → 0, k=7 → 4, k=3 → 3, k=4_000_000 → 30
- 제출 결과: AC
