# BOJ 28280 귀납법

## 1) 문제 요약
- 문제 링크: https://www.acmicpc.net/problem/28280
- 목표: 1에서 시작해 `×2` 또는 `-1` 연산을 최소 횟수로 반복해 k를 만든다
- 제약: 도중 모든 값은 양의 정수여야 한다
- 출력: 최솟값, 불가능하면 `Wrong proof!`

## 2) 문제 핵심
- T개의 쿼리, k의 총합 ≤ 4×10⁶
- 1에서 k까지의 BFS/DP로 최솟값을 구하는 문제
- "Wrong proof!" 조건: 어떤 k는 도달 불가능한가?

## 3) 관찰 포인트
- DP 를 활용하고 싶음 공간이 4*10^6 밖에 안됨
- 점화식은 대충 DP(K) = MIN(DP(K/2), DP(K-1)) + 1
- DP 풀이는 func DP(int v) int 구조로
- ret = memo[v] if ret == -1 { memo = min(DP(v / 2), DP(v -1)) + 1 } return memo
- 요련 구조로 짜줘 테스트코드도

## 4) 입출력 정리
- 입력: T, 각 줄에 k
- 출력: 최솟값 or `Wrong proof!`
- 제한: T ≤ 20, k의 총합 ≤ 4×10⁶, 시간 1초
