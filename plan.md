# BOJ 25208 새벽의 탐정 게임

## 1) 문제 요약
- 문제 링크: https://www.acmicpc.net/problem/25208
- 목표: 탐정이 한 면이 뚫린 감옥을 굴려 도둑 위치에 뚫린 면이 바닥으로 도달
- 제약: 4 ≤ N, M ≤ 500 / 테두리 전체 벽
- 출력: 최소 굴리기 횟수 (불가 시 -1)

## 2) 문제 핵심
- 감옥 = 한 면이 뚫린 정육면체, 상태 = 격자 좌표 + orientation(24가지)
- 승리: 도둑 칸에 뚫린 면(0번)이 바닥일 때 도달
- 패배: 도둑 칸에 막힌 면이 바닥일 때 → 해당 이동 스킵

## 3) 설계 결정 (discuss-phase)
- 상태: dp[x][y][s], s = north_face * 6 + bottom_face (범위 0~35, 유효 24개)
- 면 0 = 뚫린 면, 초기 큐브 = {바닥:0, 윗:1, 북:2, 남:3, 동:4, 서:5}
- 초기 s = 2*6+0 = 12
- 전환 테이블: 초기 배열 BFS로 24개 상태 × 4방향 자동 생성
- 이미 저장된 횟수 이하면 스킵 (최소값 보장, 사이클 방지)

## 4) 입출력 정리
- 입력: N M / N줄 × M문자 (D=탐정, R=도둑, #=벽, .=빈칸)
- 출력: 정수 1개 (최소 이동 횟수 또는 -1)
- 시간: 1초 / 메모리: 1024 MB

## 5) 버린 아이디어
- 최단 거리 + parity 접근: 거리만으로 orientation 판별 시도 → 경로 의존성 때문에 불가 (기각)

## 6) 구현 준비 메모
- main.go: solve() BFS 완전 구현 완료 (buildRollTable + BFS dp)
- main_test.go: 예제 3개 TestSolve 작성 완료

## 7) 테스트 메모
- 예제 1: 5×5, D=(1,1) R=(3,3) → 4
- 예제 2: 6×5, D=(4,2) R=(2,3) → 7
- 예제 3: 4×6, D=(1,4) R=(1,1) → -1 (경로 없음)
