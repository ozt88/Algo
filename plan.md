&#65279;# BOJ 31864 눈송이 탕후루 만들기

## 1) 문제 요약
- 문제 링크: https://www.acmicpc.net/problem/31864
- 목표: `(0,0)`에서 끝점 `(ex, ey)`까지의 선분 위에 있는 과일을 최대한 많이 꽂기
- 제약: N, M ≤ 300,000 / 좌표 절댓값 ≤ 10^9
- 출력: 최대로 꽂히는 과일 수

## 2) 문제 핵심
- 과일 `(fx, fy)`가 선분 `(0,0)-(ex, ey)` 위에 있는 조건:
  - 원점·과일·끝점이 **일직선** (cross product = 0)
  - 과일이 끝점보다 원점에 **가깝거나 같음** (같은 방향, 스케일 ≤ 끝점)
- N·M 최대 30만 → O((N+M) log N) 이하 필요

## 3) 관찰 포인트
- 방향 정규화: `gcd(|fx|, |fy|)`로 나눈 `(fx/gcd, fy/gcd)` pair → map key
- 거리 비교: scale = gcd 값. `과일 scale ≤ 끝점 scale`이면 포함
- 그룹화 + 카운트: map[방향] = 정렬된 scale 배열 → 끝점마다 binary search

## 4) 입출력 정리
- 입력: `N M` → N줄 과일 좌표 → M줄 끝점 좌표
- 출력: 정수 1개 (최대 꽂히는 과일 수)
- 시간: 2초 / 메모리: 512 MB

## 5) 구현 준비 메모
- main.go 스캐폴드 작성됨
- 풀이 로직: 방향 정규화(GCD) + binary search 구현 완료

## 6) 테스트 메모
- 예제 1: 9개 과일, 5개 끝점 → 3 (끝점 (4,4) 방향, dir(1,1) scale 1·2·3 포함)
