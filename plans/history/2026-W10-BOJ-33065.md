# 2026-W10-BOJ-33065

## Problem
- Source: BOJ
- Name: Observing Objects
- Link/ID: 33065

## Summary
- Goal: Count connected components of pixels with identical RGB values under 8-direction adjacency.
- What made it tricky: 8-direction connectivity and large grid size up to `1000 x 1000`.

## I/O Notes
- Input: `N M`, followed by `N` rows of `3M` integers describing RGB triplets.
- Output: A single integer, the number of objects.
- Constraints: `1 <= N, M <= 1000`, color channels in `[0, 255]`.

## Discussion Log
- Initial observations: The task is pure connectivity on a grid, with equality defined by packed RGB values.
- User-led direction: Use Union-Find by merging adjacent same-color cells.
- Rejected ideas: Full 8-neighbor scan was unnecessary because half-neighbor scan covers all undirected edges.
- Chosen approach and why: Union-Find with checks on `left`, `up-left`, `up`, `up-right` only. This preserves 8-direction connectivity without duplicate edge processing.

## Implementation Notes
- Parsing decisions: Packed each RGB triplet into one `int` using `(r << 16) | (g << 8) | b`.
- Core state / data structure: `parent[]`, `size[]`, and row-major `indexOf(m, r, c)`.
- Important edge cases: diagonal-only connectivity, single pixel image, all unique colors.

## Verification
- Sample cases checked: problem sample returned `3`.
- Additional cases checked: single pixel, diagonal merge case.
- Final result: local tests and sample run passed before moving on.

## Retrospective
- What to reuse next time: half-neighbor Union-Find pattern for undirected grid connectivity.
- What to improve: archive notes immediately after problem completion instead of after the next problem starts.
