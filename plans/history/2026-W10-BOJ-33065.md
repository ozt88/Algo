# 2026-W10-BOJ-33065

## 문제
- 출처: BOJ
- 이름: Observing Objects
- 링크/번호: 33065

## 요약
- 목표: 같은 RGB 값을 가지며 8방향으로 연결된 픽셀들의 연결요소 개수를 센다.
- 까다로웠던 점: 격자 크기가 최대 `1000 x 1000`이고, 대각선 포함 8방향 연결을 정확히 처리해야 한다.

## 입출력 정리
- 입력: `N M` 이후 `N`개의 줄에 걸쳐 각 픽셀의 RGB 값이 `3M`개 정수로 주어진다.
- 출력: 객체 개수 하나를 출력한다.
- 제약: `1 <= N, M <= 1000`, 각 색상 채널은 `[0, 255]`.

## 논의 기록
- 처음 관찰한 내용: 본질은 격자 연결요소 문제이며, 픽셀 동일성은 RGB 완전 일치로 정의된다.
- 사용자 주도 방향: 같은 색의 인접 픽셀을 Union-Find로 병합하는 방식으로 진행했다.
- 버린 아이디어: 8방향을 매번 전부 확인하는 방식은 중복 간선 처리가 많아 비효율적이었다.
- 최종 선택 방향과 이유: `좌`, `좌상`, `상`, `우상` 네 방향만 확인해도 무방향 8방향 연결을 모두 커버할 수 있어 Union-Find와 잘 맞았다.

## 구현 메모
- 파싱 결정: RGB 세 값을 `(r << 16) | (g << 8) | b` 형태의 `int` 하나로 묶어 저장했다.
- 핵심 상태 / 자료구조: `parent[]`, `size[]`, 그리고 `(r, c)`를 일차원 인덱스로 바꾸는 `indexOf`.
- 중요한 엣지 케이스: 대각선으로만 연결되는 경우, 단일 픽셀, 모든 색이 서로 다른 경우.

## 검증
- 확인한 예제: 문제 예제 결과 `3`을 확인했다.
- 추가로 확인한 케이스: 단일 픽셀, 대각선 연결 케이스.
- 최종 결과: 로컬 테스트와 예제 실행이 모두 통과했다.

## 회고
- 다음에도 재사용할 점: 무방향 격자 연결 문제에서 절반 방향만 보는 Union-Find 패턴.
- 개선할 점: 문제 완료 직후 바로 히스토리를 남겨 문맥 손실을 줄이기.
