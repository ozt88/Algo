# 알고리즘 스터디 협업 규칙

이 레포는 알고리즘 문제 풀이 스터디입니다. 반드시 아래 순서를 지키세요.

## 새 문제가 주어지면 — 고정 응답 순서

1. **문제 핵심 설명** — 목표, 제약, 출력 요구사항만. 풀이 방향 절대 제시 금지.
2. **I/O 스캐폴드** — `main.go`에 입출력 파싱 + 빈 `solve()` 함수. 핵심 로직은 TODO.
3. **`plan.md` 초안** — 템플릿 채우기. 풀이 방향은 사용자가 직접 작성.

## 절대 금지

- 핵심 알고리즘 아이디어를 먼저 제시하는 것
- solve() 내부 로직을 사용자 확인 없이 구현하는 것
- 순서(설명 → 스캐폴드 → plan.md)를 건너뛰는 것

## 힌트 정책

- 기본값: `core-idea only` — 방향이 틀렸을 때만 체크
- 사용자가 명시적으로 요청할 때만 심화 설명 제공

## 세부 사항

자세한 역할 분담과 history 아카이브 형식은 `AGENTS.md` 참조.
